Table [Help Patch]

Define a new table

Syntax

Table options

Options

    -name
    Optional user-defined name for the table
    -size
    Number of entries in table
    Type: int
    Default: 256
    -min
    Lower bound on table-lookup function
    Type: float
    Default: 0.0
    -max
    Upper bound on table-lookup function
    Type: float
    Default: 1.0
    -src
    Read table values from an external file
    Type: string
    -copy
    Copy table values from an existing table
    Type: string
    -inline
    Display the table data inline with the patch window, and allow inline editing
    Type: Boolean
    Default: false

Inlets

    None

Outlets

    None

Comments

A table in OSW is a collection of ordered floating-point numbers, which can be looked up either as elements in a traditional integer-indexed array, or as values in a floating-point function evenly spaced over the range between min and max.

OSW includes several built-in tables:

  • /sine: the function sin2px evaluated between 0 and 1 (i.e., one period of a sine function).
  • /cosine: the function cos2px evaluated between 0 and 1.
  • /exp: the function e-px evaluated between 0 and 1.
  • /sinc: the “sinc” function sin(x)/x evaluated between -2p and 2p. Similar to previous tables, this period is normalized between 0 and 1.

All of these tables have 1024 entries and are normalized to floating-point indices between 0 and 1.

Use the Table transform to define additional tables in your application. Initially, a table contains all zeroes when you define it. You can manually edit the table by double-clicking the transform. This brings up a dialog box that lets you "draw" new table values using either a line or a freehand tool:

You can also modify tables algorithmically using the table::Peek and table::Poke transforms, or store blocks of samples (e.g., from audio input) into a table using table::Record.
